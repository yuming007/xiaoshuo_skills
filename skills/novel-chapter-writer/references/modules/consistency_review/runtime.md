# 一致性审查模块运行规则

## 一、这是正文完稿后的强制流程

默认顺序：

1. 写完一章
2. 不急着交付
3. 先做六检
4. 有问题就回对应模块修
5. 六检通过后再落盘

## 二、逐章必查六项

### 1. 剧情逻辑

检查：

- 关键事件前提够不够
- 关键决定触发够不够
- 事件后果有没有真正改变局面

不稳时回：

- `plot_logic/runtime.md`

### 2. 人物目标

检查：

- 主角本章目标是否清楚
- 过程中是否无故漂移

不稳时回：

- `character_consistency/runtime.md`

### 3. 情绪与关系

检查：

- 情绪有没有来路
- 关系温度是否和上一场接得上

不稳时回：

- `character_consistency/runtime.md`
- 必要时联动 `transition/runtime.md`

### 4. 身体与信息状态

检查：

- 伤势、疲劳、秘密、误会、已知信息有没有丢

不稳时回：

- `character_consistency/runtime.md`
- `plot_logic/runtime.md`

### 5. 场景与转场

检查：

- 读者会不会迷路
- 时间、空间、视角是否清楚
- 上一场余力有没有被带到下一场

不稳时回：

- `transition/runtime.md`

### 6. 章末承接

检查：

- 章末是否收在变化上
- 下一章第一拍是否能接住

不稳时回：

- `chapter_ending/runtime.md`
- `transition/runtime.md`

## 三、最小内部审查输出模板

```text
1. 剧情逻辑：稳 / 不稳，问题在……
2. 人物目标：稳 / 不稳，问题在……
3. 情绪关系：稳 / 不稳，问题在……
4. 身体信息：稳 / 不稳，问题在……
5. 场景转场：稳 / 不稳，问题在……
6. 章末承接：稳 / 不稳，问题在……
```

任意两项不稳，不要直接交稿。

## 四、去 AI 味联动闸门

六检通过后，不代表文面就能交。

至少再问五句：

1. 有没有一句就能看出的高危模板句
2. 有没有连续两到三段在重复解释同一个意思
3. 有没有章末总结体
4. 有没有所有人物都像同一个人在说话
5. 有没有需要回读才明白主语、动作或因果的绕句

只要其中两句答案是“有”，就回：

- `anti_ai_voice/runtime.md`
- `../readability-rules.md`

## 五、续写与重写后的补查

如果本次不是全新写章，而是续写旧稿或重写旧章，六检后再补查三件事：

1. 旧伏笔状态有没有被改动
2. 时间线有没有被挤乱
3. 关键人物状态有没有因此变化

这三项不稳，就要同步回填 tracking 文件。

## 六、章纲阶段的简化用法

章纲默认至少检查：

- 结构是否清楚：回 `volume_outline/runtime.md`
- 因果是否成立：回 `plot_logic/runtime.md`
- 章末是否有半步牵引：回 `chapter_ending/runtime.md`

承接特别紧或人物特别多时，再补查：

- `transition/runtime.md`
- `character_consistency/runtime.md`
