# 连载追踪协议

## 目标

把长篇连载里最容易写错的三类信息单独留痕：

- 伏笔当前处于什么状态
- 时间线推进到哪里
- 关键人物此刻处于什么关系与风险位置

这不是额外做文档工作，而是为了减少续写时的失忆和错接。

## 最简记忆包

正文或章纲开工前，不要把所有参考都平均加载。先从追踪文件里提一份“最简记忆包”。

筛选标准只有一条：

- 如果不知道这条信息，这一章就有较大概率写错，那就保留

最简记忆包至少包含四块：

```markdown
## 最简记忆包（第N章）

### 当前位置
- 故事时间：
- 当前职分/处境：
- 当前直接危机：

### 相关人物状态
- 人物A：
- 人物B：

### 本章相关伏笔
- Fxxx：
- Fxxx：

### 本章约束
- 程序/制度约束：
- 人情/关系约束：
- 地点/时间约束：
```

## 追踪文件

### 1. `tracking/foreshadows.md`

只记仍有后效的伏笔，不记已经彻底消化掉的普通细节。

推荐格式：

```markdown
# 伏笔追踪

| ID | 内容 | 埋设章节 | 当前状态 | 预计作用阶段 | 备注 |
|----|------|---------|---------|-------------|------|
| F001 | 例 | 第4章 | 已埋 | 近程/中程/远程 | 说明 |
```

状态建议只用这四类：

- `待埋`
- `已埋`
- `推进中`
- `已回收`

### 2. `tracking/timeline.md`

只记对后续承接有影响的时间刻度，不做流水账。

推荐格式：

```markdown
# 时间线

- 当前故事时间：
- 当前阶段：

| 章节 | 时间点 | 事件 | 结果 |
|------|-------|------|------|
| 第6章 | 某日某刻 | 例 | 例 |
```

### 3. `tracking/character-status.md`

只追踪主要人物，尤其是会跨章持续起作用的人。

推荐格式：

```markdown
# 角色状态

## 人物名
- 当前身份：
- 当前立场：
- 与主角关系：
- 当前外在压力：
- 当前隐藏信息：
- 最近变化：
```

## 使用顺序

正文 skill 与大纲 skill 读取追踪文件时，优先顺序如下：

1. 只抽取和当前章有关的条目
2. 与本章无关的追踪条目不必展开
3. 如果追踪文件与已成稿正文冲突，以正文为准，并在本次完成后回填追踪文件

## 更新时机

### 正文完成后，至少检查：

- 本章是否新埋了伏笔
- 本章是否推进或回收了旧伏笔
- 本章是否改变了时间位置
- 本章是否改变了关键人物状态

### 大纲改线后，至少检查：

- 后续伏笔的预计作用阶段是否变化
- 关键人物立场与风险是否变化
- 当前直接危机的描述是否需要重写

## 历史权谋文特别提醒

- 不要把“情绪波动”当成主要追踪对象，优先追踪制度位置、关系温度、责任链和风险来源
- 伏笔描述优先写“谁借什么程序做了什么手脚”，不要写成空泛悬念词
- 时间线优先写“这件事在谁之前、在什么门、在什么差事里发生”，不是只写某日某夜
