# 命名规则

## 总原则

- 名字要服务连载阅读，不服务作者自嗨
- 第一要求是顺口、能记、能区分
- 第二要求才是仙侠味
- 读者能一眼分清“谁是谁、哪个势力干什么”，比名字华丽更重要

## 本书命名气质

- 主角线人物：
  - 偏稳、实、清
  - 少浮夸，少魔气，少戏文腔
- 家族名：
  - 要像能落地经营、能被叫出来的家族
- 势力名：
  - 更偏地方感、功能感、行业感
  - 不要全做成“九霄、太古、万劫、无上”一类泛仙侠模板词
- 地名：
  - 优先地貌感和路线感
  - 方便后续写药田、水路、矿路、坊市交易

## 人名规则

- 陆氏旁支人物优先使用：
  - 陆 + 双字名
  - 风格偏清、正、稳
- 女性角色：
  - 不要大量使用柔弱香艳词
  - 更重“清、禾、岚、宁、素、芷、澄”这类能落气质的字
- 男性角色：
  - 不要清一色“天、玄、龙、霸、战”
  - 更重“承、青、衡、岳、庭、岩、川、行”这类兼顾现实感和仙侠感的字

## 势力名规则

- 宗门执行机构：
  - 偏规则、职责感
  - 例如“契山院”这类
- 商盟：
  - 偏流通、河路、仓路、分单感
- 匠户组织：
  - 偏手艺、工事、修造感
- 巡矿或灰色组织：
  - 偏地缘、旧路、分利感

## 地名规则

- 域、坊、市、岭、河、关、渡、湾、潭、谷 都是前期高频可用后缀
- 少用过虚的大地名堆砌
- 前期一个名字最好一看就知道它更像：
  - 地方行政节点
  - 资源节点
  - 交易节点
  - 山野险地

## 工具辅助

- 需要新命名时，可用：

```bash
node ../../../../scripts/fantasy_naming_project.js --type name --count 20 --family 陆
node ../../../../scripts/fantasy_naming_project.js --type clan --count 20 --kind 社
node ../../../../scripts/fantasy_naming_project.js --type location --count 20 --kind 岭
```

- 工具只负责给候选，不负责定稿
- 最终命名必须人工筛选
- 尤其是人名和功法名，不建议直接采用第一轮输出
- 默认优先使用项目过滤版，而不是原始随机直出

## 当前项目实际筛选经验

- 陆氏族人名：
  - 工具直出时，常会出现偏轻、偏飘、偏艳的名字
  - 更适合“先抽 20 到 30 个，再挑 2 到 3 个保留”
- 势力名：
  - `会`、`社`、`院`、`坊` 这类后缀更贴本书
  - `宗`、`宫`、`神教` 容易偏模板化大派感，需慎用
- 地名：
  - `岭`、`河`、`湾`、`渡`、`潭` 一类更适合前期地面叙事
- 功法名：
  - 直出时容易过长
  - 要优先保留“功能感”，再考虑仙侠味
