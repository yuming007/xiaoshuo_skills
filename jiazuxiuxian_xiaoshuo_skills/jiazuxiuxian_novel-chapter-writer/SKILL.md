# 仙侠家族小说统一写作

## 默认工程

除非用户明确指定别的小说工程目录，否则一律使用默认共享工程：

`jiazuxiuxian_xiaoshuo_skills/jiazuxiuxian_novel-chapter-writer/assets/sample-novel-project/`

默认工程采用如下结构：

- `设定/`
- `大纲/`
- `正文/`
- `追踪/`
- `参考资料/`

其中：

- `设定/世界观/` 放世界观、修炼体系、地图、规则
- `设定/角色/` 放角色卡
- `设定/势力/` 放家族、宗门、坊市、附庸等势力文件
- `设定/关系.md` 放关系映射
- `设定/题材定位.md` 放题材卖点、对标、风格约束
- `设定/命名储备.md` 放稳定命名备选
- `大纲/大纲.md` 放全书卷级结构
- `大纲/卷纲_*.md` 放单卷结构
- `大纲/细纲_第NNN章.md` 放单章细纲
- `正文/第NNN章_章名.md` 放正文
- `追踪/上下文.md`、`追踪/伏笔.md`、`追踪/时间线.md`、`追踪/角色状态.md` 做连续性管理

如果当前仓库根目录存在：

`《家族修仙：这一脉，由我来立》小说策划案.md`

则把它视为当前默认项目的主策划案和外部总锚点。

## 人工二审覆盖层

保留人工二审覆盖机制，但不再使用历史项目那套目录。

如果小说工程根目录存在以下任一目录：

- `人工修改后的正文/`
- `正文_人工二审/`
- `人工二次审核过后的正文/`

如果同时存在多个目录，优先顺序为：

- `人工修改后的正文/`
- `正文_人工二审/`
- `人工二次审核过后的正文/`

执行规则：

- 同章节号若人工二审目录与 `正文/` 都存在，一律优先读取人工二审稿
- 人工二审稿同时视为更高优先级的 canon 和更可靠的文风锚点
- 继续写新章时，默认仍写回 `正文/`
- 只有用户明确要求时，才把新稿直接写进人工二审目录
- 如果用户在当前会话里补充了新的人工文风反馈，先把反馈提炼成 3 到 6 条可执行规则，写入 `references/manual-review-findings.md`，再继续正文、章纲或设定任务
- 这类规则必须具体到词、句式、承接方式、信息透露方式或场景写法，不能只写“更自然”“更爽”
- 如果当前工作区存在 skill 源目录，且 `~/.codex/skills/` 中也安装了同名 skill，则任务结束后要把变更同步到已安装目录

## 能处理的任务

这个 skill 以后不再只负责正文，而是统一处理三类任务：

### 1. 正文任务

常见说法：

- 写第 8 章
- 续写第 9 章正文
- 重写第 6 章，让族议更有压迫感
- 根据现有设定和前文生成第 10 章

### 2. 章纲任务

常见说法：

- 生成第 7 章大纲
- 根据前文补第 8 章章纲
- 规划第 9 章到第 12 章
- 重做第一卷后半段走向

### 3. 设定任务

常见说法：

- 补一下主角身世设定
- 新增一个宗门设定
- 整理陆家支脉关系
- 重写世界观总览
- 给第一卷补一版题材定位

## 任务分流

先判断用户要你做哪一类：

- 涉及“写章、续写、重写正文”，按“正文模式”执行
- 涉及“大纲、章纲、规划几章、卷纲、走向”，按“章纲模式”执行
- 涉及“设定、角色卡、势力卡、世界观、关系、题材定位、命名储备”，按“设定模式”执行

如果用户一次要求多件事：

- 先做最影响后续的那件
- 正文缺章纲时，先补章纲再落正文
- 章纲或正文明显依赖缺失设定时，先补关键设定，再继续主任务
- 每完成一步，都要把 `追踪/` 相关上下文写回，再进行下一步

## 默认体量

### 正文

除非用户明确指定更短或更长的字数目标，否则默认把单章正文控制在 3000 字左右。

执行口径：

- 默认目标区间为 2800 到 3300 字
- 非特殊情况不低于 2600 字，不高于 3600 字
- 低于 2600 字时，优先补足事务、阻力、兑现和章末牵引
- 高于 3600 字时，优先压缩重复解释、合并弱场景，必要时把次要信息后移到下一章

### 章纲

除非用户明确要求短章或大章，否则默认按“可支撑约 3000 字正文”倒推章纲密度。

- 单章通常准备 3 到 5 个有效场景
- 或 2 个大场景加 1 到 2 个过渡场景
- 不要薄到逼正文注水
- 也不要满到把正文挤成流水账

### 设定

设定默认以“够当前连载落地”为准：

- 直接服务当前章或当前阶段矛盾
- 能写进场景、动作、关系和资源事务
- 不为了显得宏大而堆空概念

## 信息优先级

遇到信息冲突时，严格按下面优先级处理：

1. 工程内 `人工修改后的正文/`、`正文_人工二审/` 或 `人工二次审核过后的正文/` 中存在的对应章节
2. 工程内 `正文/` 中已成稿正文
3. 工程内 `大纲/细纲_第NNN章.md`
4. 工程内 `大纲/卷纲_*.md`
5. 工程内 `大纲/大纲.md`
6. 工程内 `追踪/上下文.md`、`追踪/伏笔.md`、`追踪/时间线.md`、`追踪/角色状态.md`
7. 工程内 `设定/` 下的相关文件
8. 仓库根目录的 `《家族修仙：这一脉，由我来立》小说策划案.md`
9. skill 内的 `references/current-story-state.md` 与 `references/novel-bible.md`
10. skill 内的通用写作约束和平台规则
11. 一般性仙侠、家族修仙、连载文默认推断

不要让参考设定反过来覆盖已成稿正文。

```bash

```

```bash

```


### 2.1 最简记忆包

开工前必须先压成一份简短的“最简记忆包”，至少包含：

- 当前故事时间与当前直接危机
- 当前关键人物状态
- 当前相关伏笔
- 当前资源、家族、宗门、时间和关系约束

判断标准只有一条：

- 如果不知道这条信息，这次任务就有较大概率写错，那就保留

和当前任务无关的设定、人物、远线伏笔不要硬塞进当前上下文。

### 2.2 信息权限检查

正文和章纲都要额外检查：

- 主视角人物此刻知道什么，不知道什么
- 新信息准备通过什么渠道到手
- 哪些判断必须等对白、物证、站位、称呼、现场反应出来后才能成立

如果提前把人物“已经看懂一切”写满，正文就容易变成解释腔。

### 2.3 补充网文写法规则

- 场景落笔前，先在脑中过一遍五感；真正写时，只挑和当前事务、危险、身份、资源状态直接相关的两到三种感官落笔，不追求每场都写满
- 正文默认至少八成信息贴主视角人物的体感、判断和利害反应来写，少写脱离人物感受的“事物本身如何”；主角高光、旁人被震住或要制造信息差时，可以短切配角视角；只有必须交代关键认知差时，才短用上帝视角
- 写伤痛、惊惧、疲惫、寒热、窒息等反应时，顺序默认是“底层生理反应 -> 情绪浮现 -> 人物确认自己在怕什么、痛什么、慌什么”，不要一上来先下抽象判断
- 平时塑造人物，先给年龄、体态、衣着、职业动作和当下状态；到生死冲突、重大得失或立场翻脸时，再深挖人物的执念、恐惧、价值排序和关系取舍
- 危机场景默认把结果后置，先写过程、异样、阻力和代价，让读者先感到事态不对，再揭开结果；不要一上来把答案讲完
- 节奏上默认压缩低张力的走路、转场、重复解释和无效寒暄，把篇幅留给冲突、反转、兑现和高光；高光处要慢一拍，把关键动作、旁人反应和局势变化拆开写
- 写主角翻盘、立威、逆转时，先用旁人判断、局势悬殊、代价预估去抬高“不可能”；写厄运逼近、坏事落地时，先给人物一个短暂的正常预期或侥幸，再用反打把坏结果落下来

## 正文模式

### 1. 缺章纲时的处理

如果当前章没有明确章纲：

- 直接用本 skill 先补章纲，再回到正文
- 如果用户就是要你直接写，也要先在心里生成一个简要场景提纲，再落正文

不要在毫无章纲和前文承接判断的情况下直接硬写。

### 2. 续写与重写分流

如果用户说“续写第 N 章”：

- 先读旧稿
- 判断是接在旧稿末尾继续写，还是旧稿虽然存在但需要整章修
- 若旧稿已经形成完整章结构，但只差后半段，则优先接续
- 若旧稿方向、节奏、承接已经歪掉，不要硬续，改按重写处理

如果用户说“重写第 N 章”：

- 先读旧稿
- 必读前一章正文
- 若存在后一章，也必读后一章正文，避免重写后直接撞坏承接
- 重写后必须更新 `追踪/` 四件事

### 3. 正文写法

正文默认服务于起点男频仙侠 / 家族修仙 / 势力经营取向。

保留这些特征：

- 第三人称近距离视角，核心感知紧贴当前主视角人物，默认以陆承前为主
- 默认以主视角承接大部分信息，主角高光或信息差明显处可短切配角视角，但切换必须短、准、能服务当段爽点或悬念
- 章节标题格式统一为“第X章 章名”，默认纯文本单独成行，不额外加 Markdown `#`
- 开头常用明确时间、地点、场景锚点切入
- 开头第一拍优先落“谁在何处做什么”
- 环境描写要更具体，但只写和当章事务、冲突、身份、资源状态直接相关的那一层；内心复盘仍要压量
- 每个主场景尽量落一到两个可触可感的环境锚点，优先写光线、湿度、气味、材质、器物状态和地面细节
- 重要场景下笔前默认先过一遍五感，再筛出最有效的两到三种感官写进动作、对白和视线移动里，不平均铺陈
- 场景不要整段堆砌，优先拆进人物动作、对话和视线移动里，让环境跟着事务一起出现
- 重要人物首次出场、换场合或身份压迫明显时，补一层服装与装束细节，服务身份、阶层、劳动状态和战斗准备
- 动作描写优先写起势、发力、受力、落点和余波，不只写结论
- 关键动作尽量拆成三到五拍；情绪复杂处优先补“二次动作”或短停顿，不要直接用旁白替人物解释
- 写疼痛、恐惧、疲惫、窒闷等反应时，先落生理变化，再落情绪，再落人物对情绪的认知，避免“先知道自己害怕，后补身体反应”的假感
- 术法、武技、血脉异象和护体手段，优先写气机怎么起、借了什么媒介、形态怎么成、周遭环境和旁人怎么被影响
- 以上场景、动作、服装、术法描写要求属于默认硬规则，不需要用户额外提出才执行
- 对话要体现试探、权衡、利害、家族位置与资源焦虑
- 主角思考方式始终偏现实：先看资源、规矩、利益、执行条件，再看名分和感情
- 若人工稿已经把主角声线调成更口语、更现代的即时反应，默认沿用，不要退回过分端着的旧书面腔
- 允许短促吐槽、反问和一点黑色幽默，但只能服务当前处境判断，不能把严肃场面写成段子
- 陆家内部称呼默认统一：陆昭衡等那一边写“主家”，陆承前这一边写“支脉”；只有在宗谱、册契、制度分类或外部正式规则里，才把“主支 / 旁支”当通类旧称使用
- 需要补世界规则时，优先从主角此刻为什么得算这笔账、为什么不能退来解释，多用资源链、势力链、名分链，少写脱场景讲义
- 人物和局势可以先给主角的一手判断，再用对白、动作和后续调查补证据
- 文气可以沉稳、克制，但可读性优先
- 优先用常用词、明白句、短中句
- 不要把章节写成设定讲义
- 危机段落默认结果后置，先写过程和异样，再揭结果；高光段落默认放慢一拍，把翻盘的关键动作、对手失态和旁人反应写够
- 结尾尽量用动作、场景余波、半句话头收住
- 家族修仙文的爽点主要来自：立规矩、保祖地、争灵田、稳药泉、拿矿脉、收客卿、养族人、补祖谱、抗宗门、扩势力
- 调查、对簿、查账、争地、族议、拍卖、守泉、守矿、宗门压价类场景，不能砍掉证据链、利益链和执行链上的关键一环

### 4. 平台节奏校准

默认按起点男频仙侠 / 家族修仙 / 经营流处理：

- 第一章先让事发生，再补设定
- 第二章先把身份和关系落稳，再补规则
- 第三章必须让主角做出第一轮明确动作
- 每章至少要有：当章事务、当章阻力、部分兑现、章末牵引
- 主角必须在具体处境里使用自己的优势，不能只在脑中高明
- 追读感主要来自问题清楚、事务推进和章末余力

### 5. 落盘规则

默认写入：

- `正文/第NNN章_章名.md`

如果当前章原文件名已存在并带章名：

- 优先覆盖原文件

如果当前章只有 `第NNN章_待定.md`：

- 正文完成后可以按新章名重命名

## 章纲模式

### 1. 章纲必须解决什么

每一章的大纲都要解决四件事：

- 这一章承接了什么
- 这一章真正推进了什么
- 这一章制造了什么新的阻力或代价
- 这一章结尾把读者推向哪一章

不要只写主题词，不要只写人物情绪。

同时注意：

- 每章尽量包含“当章事务、当章阻力、部分兑现、章末钩子”四个要素
- 仙侠家族文的冲突必须合资源、合境界、合势力、合关系、合当下处境
- 钩子优先落在祖谱、灵脉、灵田、药泉、矿脉、供奉、婚盟、附庸、客卿、族规和突破节点上
- 优先写“人怎么做、别人怎么反应、信息怎么到手”
- 关键高光场景要提前设计“悬殊感”或“做不到”的预期来源，想清谁先觉得主角撑不住、代价在哪里、翻面后谁先失态
- 关键危机场景要优先设计“正常预期 -> 异样累积 -> 坏结果揭开”的顺序，不要在大纲里先把答案说死

### 2. 开局章纲规则

如果处理的是第 1 章到第 3 章，或某一卷的卷首章，必须严格满足下面分工：

- 第 1 章：穿越、危机、身份反差、家族将散、祖谱异动中至少出现两项
- 第 2 章：快速落身份、关系、生存环境，用具体场景补信息
- 第 3 章：让主角做出第一轮明确决断，并推出下一阶段路径

不要把前三章都写成设定说明书。

### 3. 多章规划

如果用户一次要求规划多章，不要把每章都写成各自独立的小冲突。

至少先确定：

- 这几章共同推进的阶段矛盾是什么
- 哪一章负责压局
- 哪一章负责翻面
- 哪一章负责兑现阶段结果
- 哪条伏笔在这几章内推进，哪条只埋不收

### 4. 大纲格式

优先使用下面格式，确保正文模式可直接拿来写：

```markdown
## 第008章 章名

- 本章定位：
- 本章即时看点：
- 核心冲突：
- 承接前文：
- 推进结果：
- 本章兑现：
- 本章资源变化：
- 本章家族进度：

### 场景一
- 地点：
- 出场人物：
- 场景目标：
- 关键事件：
- 结果：

### 场景二
...

- 本章伏笔回收：
- 本章新伏笔：
- 章末钩子：
```

连续多章时，在第一章前额外补：

```markdown
## 本组章节阶段说明

- 阶段矛盾：
- 阶段推进目标：
- 阶段兑现点：
- 本组共用伏笔：
```

### 5. 落盘规则

默认写入：

- `大纲/细纲_第NNN章.md`

处理原则：

- 已有该章细纲：整体替换
- 没有该章细纲：新建该文件
- 用户要求连续生成多章：按顺序一次性新建或替换多个 `细纲_第NNN章.md`

如果用户要求调整全书结构、卷结构或明显改线，还应同步修订：

- `大纲/大纲.md`
- 对应 `大纲/卷纲_*.md`
- `追踪/上下文.md`
- `追踪/伏笔.md`
- `追踪/角色状态.md`

## 设定模式

### 1. 设定写法

设定默认服务当前连载，不服务空转炫设定。

执行原则：

- 先解决当前章、当前卷、当前阶段真正要用到的问题
- 设定必须能落到人物权限、资源结构、势力关系、执行难点或可见场景
- 世界观可以铺开，但不能写成泛泛设定说明书
- 若人工稿已经在正文里补出更直给、可落地的世界规则表述，要及时拆回对应 `设定/世界观/*.md`，不要让有效口径只留在正文里
- 主角身世、家族历史、势力规则等内容，优先写成可直接被正文调用的版本
- 若已有正文已经落地某个事实，设定文件只能补足和整理，不能反改 canon

## 连续性硬约束

所有模式都必须承认以下既成事实：

- 当前项目不是历史文，而是仙侠家族 / 势力经营文
- 主角为现代小区保安队长穿越而来，核心优势是组织、止损、盯执行、看人下菜碟
- 主角所在的是青岚域临河坊陆家支脉，陆家主家长期压着这条支脉
- 核心机制是祖谱残卷 + 族运修行
- 第一卷主问题是：支脉将散、祖地将失、执谱权未定、原身死因可疑
- 长期主线不是单纯修炼，而是“从支脉到正统、从散脉到仙族”

## 关于自然度与可读性

目标是写得更自然、更像连载稿，减少模板腔和明显机器感。

默认先满足一条：

- 普通网文读者第一次读时，基本不用回看句子，就能知道谁在做什么、这段话为什么要写

不要做的事：

- 不要为了“像小说”而每段都端着
- 不要为了“高级”而密集堆砌排比、长句、抽象判断
- 不要把简单意思故意写重、写古、写得半文半白
- 不要连续两到三段只靠气氛、心理、感慨撑住，而没有新动作、新信息、新对话
- 不要专门朝“规避检测”方向设计文本

要做的事：

- 多用常见词和直给句，先把意思说清
- 让信息落在人物、动作、场景和关系里
- 每一到两段至少给读者一个可见动作、对白或新信息
- 接受局部朴、局部粗、局部留白
- 保留已有正文已经形成的自然节奏，但文面默认向更顺口、更易读校正，而不是机械继承旧稿的沉重句法
